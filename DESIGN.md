# Design System: GrantIQ

This file is the single source of truth for any future GrantIQ screen —
generated by an AI, sketched by a designer, or typed by an engineer. It
encodes anti-slop rules tuned for **B2B SaaS serving nonprofit grant
writers**, not for agency portfolios or consumer-creative products.

**If you're about to ship UI that contradicts anything below, stop and
re-read the atmosphere.**

---

## 1. Visual Theme & Atmosphere

**Density: 4 / 10** — "Daily app balanced." Enough information on
screen to feel substantive (grant tables, pipeline kanbans, readiness
scorecards), but never dense enough to overwhelm an executive director
who opened the tab between board meetings.

**Variance: 3 / 10** — "Predictable, slightly off-center." Trust > flair.
Symmetric hero layouts are fine here (this is a financial decision
surface, not a fashion brand). Asymmetric bento grids are banned on
user-facing primary flows — they introduce cognitive load our audience
doesn't pay us for.

**Motion: 3 / 10** — "Static restrained." Hover shifts, fade-ins, and
small scale transforms on active state only. No scroll-pinned reveals,
no cinematic choreography, no parallax. The product does
compliance-audit-level work; the UI should feel competent, not
theatrical.

**Overall atmosphere:** A warm, well-lit accounting office that happens
to run on AI. Calm, confident, unfussy. The interface disappears so
the user can focus on winning the next $250K grant.

## 2. Color Palette & Roles

All tokens live in `src/app/globals.css`. Use the semantic name, never
the raw hex.

### Primary surface tokens

| Role | Token | Hex | Usage |
|---|---|---|---|
| Canvas background | `warm-50` | `#FAFAF9` | page background, large surface |
| Pure surface | `white` | `#FFFFFF` | cards, modals, elevated containers |
| Soft surface | `warm-100` | `#F5F5F4` | hover states, input backgrounds, subtle zones |
| Hairline border | `warm-200` | `#E7E5E4` | dividers, card borders, input borders |
| Structural border | `warm-300` | `#D6D3D1` | inputs in focus-adjacent state, heavier dividers |

### Text tokens

| Role | Token | Hex | Usage |
|---|---|---|---|
| Primary text | `warm-900` | `#1C1917` | headings, primary body, confident statements |
| Body text | `warm-700` | `#44403C` | long-form paragraph copy, feature descriptions |
| Secondary text | `warm-500` | `#78716C` | metadata, timestamps, tertiary labels |
| Muted text | `warm-400` | `#A8A29E` | placeholder, disabled, "last updated" crumbs |
| Icon default | `warm-600` | `#57534E` | icons that are not brand-colored |

Use **warm-500** for "secondary" — never use a cooler gray. The palette
is warm-stone across the board.

### Accent — one color, two variants

| Role | Token | Hex | Usage |
|---|---|---|---|
| Brand accent (surfaces) | `brand-teal` | `#0D9488` | CTA fills, selected states, brand marks |
| Brand accent (text) | `brand-teal-text` | `#0B7C72` | links, accent text on white (WCAG AA 4.5:1) |
| Brand accent dark | `brand-teal-dark` | `#115E59` | hover state of brand-teal buttons |
| Brand accent light | `brand-teal-light` | `#5EEAD4` | very subtle tints, tooltip backgrounds |

**The two-variant rule**: on white backgrounds, `brand-teal` (#0D9488)
fails WCAG AA for body text (3.5:1). Use `brand-teal-text` (#0B7C72)
for anything text-like. Reserve `brand-teal` for fills, icons >24px,
and large bold headings.

### Semantic status (use sparingly)

| State | Color | Hex | Rule |
|---|---|---|---|
| Success | emerald-600 | `#059669` | awarded grants, confirmations, positive-outcome stats only |
| Warning | amber-600 | `#D97706` | due-soon deadlines, "needs review", soft-gate states |
| Destructive | red-600 | `#DC2626` | overdue items, hard errors, blocked verdicts |
| Info | brand-teal-text | `#0B7C72` | neutral informational notices — do not use a separate blue |

**BANNED** in this palette:
- No purple, no indigo, no neon glow, no gradient accents
- No "AI saturated cyan/violet" — our brand is warm teal, not tech-corp
- No pure black (`#000000`) — use `warm-900` (#1C1917)
- No pure white text on non-white backgrounds — use `warm-50`

## 3. Typography Rules

GrantIQ's fonts are wired in `src/app/layout.tsx` via `next/font/google`
(Geist + Geist Mono). Do not import additional fonts without opening a
design conversation first — more fonts = more load time + style drift.

### Active fonts

| Role | Font | Loaded as |
|---|---|---|
| Display + body | **Geist** | `--font-geist-sans` (CSS variable) |
| Monospace | **Geist Mono** | `--font-geist-mono` |

### Scale — marketing pages (`clamp`, fluid)

```css
h1: clamp(2.25rem, 4vw, 3.75rem)   /* 36–60 px */
h2: clamp(1.75rem, 3vw, 2.5rem)    /* 28–40 px */
h3: 1.5rem                          /* 24 px */
body-lg: 1.125rem (18 px) / 1.75 leading
body:    1rem     (16 px) / 1.6 leading
caption: 0.875rem (14 px) / 1.5 leading
micro:   0.75rem  (12 px) / 1.4 leading
```

### Scale — app UI (fixed rem, no fluid)

```css
h1: 1.5rem    (24 px)
h2: 1.125rem  (18 px)
h3: 1rem      (16 px) semibold
body: 0.875rem (14 px) / 1.5 leading  ← app default
caption: 0.75rem (12 px) / 1.4 leading
micro: 0.6875rem (11 px)
```

**Tracking:** `tracking-tight` (-0.01em) on all headings 20px+.
Body text is default tracking.

**Weights used:** 400 (regular), 500 (medium), 600 (semibold), 700
(bold). Nothing lighter than 400 — thin weights read as fragile on a
trust product.

**Line length:** cap at `max-w-prose` (65ch) for body paragraphs.
Wider than that, eye-tracking degrades.

### Banned — typography

- **No Inter, DM Sans, Plus Jakarta Sans, Outfit, Space Grotesk** — all
  on the impeccable reflex-reject list, all scream "AI SaaS template"
- **No generic serifs** (Times, Georgia, Palatino, Garamond) anywhere
  in the app — serifs are banned in dashboards outright, and we don't
  have a magazine/editorial surface to justify them on marketing
- **No gradient text** (`background-clip: text` + gradient) — explicit
  impeccable absolute ban
- **No all-caps body copy** — reserve uppercase for short pill labels
  and eyebrow `tracking-[0.2em]` kickers
- **No thin weights on dark backgrounds** — adds cognitive strain

### Number rule

Prices, token counts, grant amounts, scorecards, and any comparable
number rendered in table rows must use `tabular-nums`. When density
exceeds 6 (admin dashboards, pipeline tables), switch to **Geist Mono**
entirely for numbers — not just `tabular-nums`.

## 4. Hero Section (Marketing Only)

GrantIQ's hero is **centered, not asymmetric**. This is a deliberate
deviation from the impeccable default. Rationale: our audience is
risk-averse (nonprofit EDs evaluating a $79-$499/mo tool), and centered
layouts signal "legitimate B2B SaaS" in their mental model. Asymmetric
hero would register as "edgy startup" — wrong trust signal.

Hero requirements:
1. **Eyebrow pill** — `bg-brand-teal` with white uppercase text,
   `tracking-[0.2em]`, 12 px size, icon at 16px. Communicates category
   (e.g., "AI-POWERED GRANT DISCOVERY & READINESS").
2. **H1** — 4-8 words, one line on desktop, tighter-than-normal leading
   (1.05). Include one teal-accented word to break visual monotony.
3. **Sub-deck** — one sentence, ≤28 words, body-lg size, warm-600.
   Must answer "what does this do for me?" in plain English.
4. **Two CTAs** — primary (filled brand-teal) + secondary (outline
   warm-300). Minimum height 48px (`h-12`), minimum horizontal padding
   `px-7`. Both must be `size-base` or larger.
5. **Trust row** — 4 micro bullets with brand-teal check icons,
   12px size, warm-500 color. Sets frictionless tone.

**Banned in the hero:**
- No inline image typography (photos between words) — off-brand
- No "scroll to explore" / bouncing chevron / filler CTA text
- No animated gradients, no particle fields, no Three.js backgrounds
- No video backgrounds — loads weird on nonprofit office networks

## 5. Component Stylings

### Buttons
- **Size scale:** `xs h-6`, `sm h-7`, `default h-8`, `lg h-9`,
  `hero h-12` (marketing override; never use h-12 in-app)
- **Corner radius:** `rounded-lg` (0.5rem) for all sizes
- **Primary variant:** `bg-brand-teal text-white`, hover
  `bg-brand-teal-dark`. Active state: `translate-y-px scale-[0.98]`
  (tactile push).
- **Outline variant:** `border-warm-300 bg-white text-warm-900`, hover
  `bg-warm-100`. Never use a brand-colored outline button — the teal
  outline reads cheap.
- **Ghost variant:** `hover:bg-warm-100`, no border. Used in nav + list
  rows.
- **Destructive variant:** `bg-red-50 text-red-700 border-red-200`,
  hover `bg-red-100`. Never solid red fill — too aggressive for our
  audience.
- **Transitions:** `transition-[color,background-color,border-color,box-shadow,transform,opacity] duration-150`
  — NEVER `transition-all` (animates layout props, flagged by impeccable)

### Cards
- **Corner radius:** `rounded-xl` (0.75rem) default, `rounded-2xl` for
  hero/feature cards
- **Border:** 1px `border-warm-200` — never thicker than 1px on default
  cards. Selected/highlighted state uses `ring-2 ring-brand-teal/20` +
  `border-brand-teal`
- **Shadow:** `shadow-sm` on hover only. No default shadows on list
  cards — the page becomes a snow-globe of shadows.
- **Padding:** `p-6` (1.5rem) default, `p-4` for dense tables, `p-8`
  for hero-tier content
- **Nested cards:** BANNED. If you need two levels, use border-top
  dividers or `bg-warm-50` inner surfaces.

### Inputs / Forms
- **Height:** `h-10` (40px) default — large enough for touch targets,
  not so large they feel consumer
- **Border:** `border-warm-300` default, `border-brand-teal` on focus,
  `ring-2 ring-brand-teal/30` on focus
- **Label position:** ABOVE input, `text-sm font-medium text-warm-700`,
  8px gap to field. No floating labels.
- **Helper text:** below field, `text-xs text-warm-500`
- **Error text:** below field, `text-xs text-red-600`, with
  `aria-invalid` on the input

### Tables
- **Divider pattern:** `border-b border-warm-200` between rows. No
  zebra striping — looks dated.
- **Hover:** `hover:bg-warm-50` on interactive rows only
- **Header:** `text-xs font-medium uppercase tracking-wide text-warm-500`
- **Numbers in cells:** `tabular-nums` or Geist Mono (see §3)

### Badges / Pills
- **Size:** `text-xs px-2 py-0.5 rounded-full`
- **Semantic fills:** bg-{color}-100 + text-{color}-700. For dark mode:
  `bg-{color}-900/30 text-{color}-400`
- **Uppercase eyebrows:** `tracking-[0.2em] uppercase text-xs
  font-semibold` — used sparingly on marketing, never in-app

### Empty states
A composed illustration or structured layout, never just "No data."
Every empty state answers: **what is this?** + **what do I do to fill
it?** Copy should be actionable ("Add your first grant to the
pipeline" > "No items").

### Loading states
**Skeletal loaders** matching actual layout dimensions — NOT circular
spinners. Use `bg-warm-100 animate-pulse rounded` blocks that
approximate the content shape. Spinners allowed only inside buttons
(`Loader2` icon) and for <2s async actions.

## 6. Layout Principles

### Container strategy
- **Marketing pages:** `max-w-6xl mx-auto px-4 sm:px-6 lg:px-8`
- **App pages:** `max-w-6xl px-4 md:px-6 py-6` (consistent with existing
  admin + dashboard layouts)
- **Dense data views** (pipeline, matches, library): `px-4 md:px-6`,
  no max-width cap — let tables breathe on wide monitors

### Grid systems
- **Prefer CSS Grid over Flexbox math.** Avoid `calc(33.333% - Xpx)`.
- **Responsive cards:** `grid-cols-1 sm:grid-cols-2 lg:grid-cols-3` is
  the default 3-column pattern. For pricing (4 tiers + enterprise),
  use `lg:grid-cols-4` + a separate enterprise row.
- **The 3-equal-cards feature row IS allowed on marketing.** This is a
  deliberate deviation from the skill — trust B2B products do this
  pattern because it works. Variety inside each card (different icon
  color, different copy length) prevents it from feeling template-y.
- **Container queries** for components that appear in both sidebar +
  main column (cards, nav items). Viewport queries for page layout.

### Spacing scale (4pt system)
```
space-1: 0.25rem (4px)
space-2: 0.5rem  (8px)
space-3: 0.75rem (12px)
space-4: 1rem    (16px)
space-5: 1.25rem (20px)  — rare, use 4 or 6
space-6: 1.5rem  (24px)  ← default card padding
space-8: 2rem    (32px)
space-10: 2.5rem (40px)
space-12: 3rem   (48px)
space-16: 4rem   (64px)  ← major section spacing
space-20: 5rem   (80px)  ← marketing section spacing
space-24: 6rem   (96px)  ← marketing major-section spacing
```

Use `gap` over margin for sibling spacing. No margin collapse hacks.

### Full-height sections
Use `min-h-screen` or `min-h-[100dvh]` — NEVER `h-screen` (iOS Safari
URL-bar jump).

### Overlap rule
**No absolute-positioned content stacking** except for:
- Tooltips / popovers
- Toast notifications
- Dropdown menus
- Hero eyebrow pill offset above an H1 (classic SaaS pattern, fine)

No visual element should occlude text content.

## 7. Responsive Rules

### Breakpoints (Tailwind defaults)
- `sm`: 640px — phone landscape / small tablet
- `md`: 768px — tablet portrait
- `lg`: 1024px — tablet landscape / small laptop
- `xl`: 1280px — laptop / standard desktop
- `2xl`: 1536px — large desktop

### Mobile-first rules
1. **Below 768px:** all multi-column layouts collapse to 1 column.
2. **Touch targets:** every interactive element ≥ 44px tap area. App
   buttons at `h-8` (32px) still need `py-2` padding to reach 44px.
3. **Horizontal overflow:** critical failure. Test pricing table,
   matches page, pipeline kanban at 375px width on every change.
4. **Type scales:** headings use `clamp()` (marketing only). Body text
   stays fixed at 14-16px.
5. **Navigation:** desktop horizontal → mobile drawer menu at `md`.

### iOS Safari traps
- Always `100dvh`, not `100vh`, for full-height sections
- Form inputs ≥ 16px to avoid auto-zoom
- `-webkit-tap-highlight-color: transparent` on interactive elements

## 8. Motion & Interaction

GrantIQ is a **motion-3** product. Default to restraint.

### Allowed motions
- **Hover:** `transition-colors duration-150` on interactive elements
- **Press:** `active:translate-y-px active:scale-[0.98]` on buttons
  (tactile push)
- **Fade-in:** on page load, 200ms fade + 4px translate-y for primary
  content blocks. Use `animate-in fade-in slide-in-from-bottom-2`
  (tw-animate-css)
- **Skeleton pulse:** `animate-pulse` on loading placeholders
- **Progress bar fills:** `transition-[width] duration-500` — width IS
  a layout property, but progress bars have bounded parents so cost is
  capped

### Banned motions
- `transition-all` — explicitly flagged by impeccable detector
- Animating `height`, `top`, `left`, `margin`, `padding` on anything
  unbounded (layout thrash)
- Spring physics libraries (Framer Motion) outside of occasional hero
  illustration — increases bundle weight for marginal perceived quality
- Scroll-pinned sections, parallax, horizontal scroll journeys
- Custom cursors
- Bounce/elastic easing (dated)
- Infinite loop animations outside of skeleton loaders

### Reduced motion
```css
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```
Already enforced globally. Do not override per-component.

## 9. Accessibility (non-negotiable)

- Contrast ≥ 4.5:1 for body text, ≥ 3:1 for large text and icons
- Use `brand-teal-text` (#0B7C72) for accent TEXT on white, not
  `brand-teal` (#0D9488 fails AA for text)
- Every interactive element has a visible focus ring: `focus-visible:ring-2 focus-visible:ring-brand-teal/50`
- Icons that convey meaning have `aria-label`; decorative icons get
  `aria-hidden="true"`
- All images have `alt` — empty string if decorative
- Forms: `aria-invalid` on error, helper/error text linked via
  `aria-describedby`
- Tables have `<caption>` or aria-label

## 10. Anti-Patterns (Banned — match and refuse)

These are the AI-slop tells that instantly tank credibility. Never
ship:

### Color
- Purple, indigo, or neon accent colors
- Gradient text (background-clip: text + gradient)
- Pure black (#000) or pure white (#fff) anywhere
- Cyan-on-dark "AI palette"
- Saturation above 80% on any accent

### Typography
- Inter, DM Sans, Plus Jakarta Sans, Outfit, Space Grotesk
- Generic serifs (Times, Georgia, Garamond, Palatino)
- Monospace headings as "technical vibes"
- Large icon with rounded corners above every heading
- All-caps body text
- Flat hierarchy (heading-subheading-body all similar sizes)

### Layout
- Nested cards (card inside card)
- Glassmorphism blur backdrops used decoratively
- Border-left or border-right greater than 1px as a colored accent
  stripe (impeccable absolute ban)
- "3 equal cards with icon + heading + text" repeated endlessly (the
  only exception: marketing feature rows, handled with variation)
- The hero metric layout template (giant number + small label + gradient
  accent)
- Centering every layout (forfeits hierarchy)

### Motion
- transition-all
- Animating height, top, left, padding, margin
- Bounce / elastic easing
- Scroll-pinned reveals, parallax
- Spring physics on every interaction
- Custom mouse cursors
- Particle fields, animated gradients
- Video backgrounds

### Copy
- "Elevate", "Seamless", "Unleash", "Next-generation", "AI-powered"
  (ironic — we ARE AI-powered, but the word is dead)
- Filler CTAs: "Scroll to explore", "Swipe down", "Learn more"
- Round-number stats ("99.99%", "10x faster")
- Testimonials without names + role + org
- Generic placeholder names (John Doe, Acme, Nexus)
- Emojis anywhere in the product UI

### Social-proof
- Logos without context ("As featured in" with random TechCrunch logo)
- Aggregate metrics without denominator ("5M users" — out of what?)
- AI-generated testimonial avatars (real photos or SVG monograms only)

## 11. Applying This Document

### When writing new code
1. Reach for tokens, not raw colors (`bg-warm-50` not `#FAFAF9`)
2. Check §10 before shipping — any match = stop and redo
3. If a visual pattern isn't covered here, prefer restraint over
   invention. Ask before adding a new variant.

### When an AI generates UI for GrantIQ
Pass this entire `DESIGN.md` into the prompt. The anti-patterns in §10
are the most important part — they stop the model from reaching for
its training-data defaults.

### When the design conflicts with this doc
Either update this doc deliberately (with a note explaining why) or
fix the design. Don't let drift accumulate silently.
